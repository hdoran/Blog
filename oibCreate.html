How To Estimate Response Probabilities Using PACER

Published January 17, 20225 min read

Response Probabilities

Response probabilities, or RP values, are used in psychometrics in multiple ways. Sometimes they are used in the Bookmark standard setting procedure to create an ordered item booklet (OIB) and other times they are used for mapping items to a reporting scale. The PACER software program can estimate the theta (or ability) associated with a specific response probability for a set of items and return a table with the RP values for each item, a table ordering all items by the RP value, and a collection of visual displays that might be useful to examine the items.

The items can be all binary, all polytomous, or a mixture of the two. All binary items are treated as a special case of the 3-parameter logistic model and the polytomous items must be the generalized partial credit model, or a special case of that model (e.g., Master's Partial Credit). The graded response model is coming soon.

In this blog post I will show how to use PACER to estimate the RP values and examine a set of visual displays using data that can be accessed here.

Preparing the Data

The data used in this tutorial can be downloaded by clicking here . The data must look like this sample:

Here we see the variable names ITEM_ID, SCOREPOINTS, and then P0, P1, P2, ..., P5. For binary items (SCOREPOINTS =1), the columns mean P0 = a-parameter, P1 = b-parameter, and P2 = c-parameter. For polytomous items, the columns have a slightly different meaning. When SCOREPOINTS > 1, then P0=a-parameter, however, the columns P1 through P5 are the step values indicating where the curves of the GPCM intersect.

Some IRT software programs represent the GPCM as \((\theta - (\delta_j - b_{j,k}))\) where \(\delta_j\) is a location parameter and \(b_{j,k}\) is a category and \(\sum_{k=1}b_{j,k}=0\). In PACER, simply transform this to be \((\theta - \lambda_{jk})\) where \(\lambda_{jk} = (\delta_j - b_{j,k})\).

Running the Analysis in PACER

Now with our data formatted, we can easily run the analysis using PACER. First, click the "Create OIB" tab in the top navigation bar and you'll end up at this screen where you simply read in the data as you would any file using browse to find the file.

You know your data are properly read in when you see "Upload Complete". Now, all you need to do to run the analysis is move over to the "Options" tab and click "Get Response Probabilities". The RP is set to .67 by default but you can change that to any value. The inputs for mean and variance are to estimate impact data from a theoretical normal distribution using those two parameters.

When you click "Get Response Probabilities" a beautiful data table prints to the screen as in the example below. This output shows the RP value, the item ID, the location of where the RP value is in the table, and the impact data representing the percentage of examinees that would score at or below this point. Notice that binary items have one location only (Step 1 of 1) but polytomous items have one row for each step parameter. So, if an item has multiple score points it will appear in multiple places in the table with the RP value needed to score in that category.

The table is searchable using the box to the right. Simply start typing in text or numeric values and the table will begin to filter as you type. Additionally, you can click any of the buttons at the top to export the table in any of the formats listed. Note, if you filter the table, it prints the filtered version. Also, note that these buttons will print the RP value only to 3 digits. If you prefer greater precision, click the "Export RP Table button" on the sidepanel to export a table with more precision than 3 digits.

Now if you head over to the "Visual Displays" tab, you can use the drop down menu and view plots for any of the items included in the analysis. If the plots seem too small, use the "Panel Size" input on the sidepanel to increase (or decrease) the size of the graphic.
