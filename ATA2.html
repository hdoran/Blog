Automated Test Assembly: Expanded Example

Published July 14, 202215 min read

Automated Test Assembly: Example 2

Someone asked how many constraints PACER can implement in its ATA module. The answer is as many of you can create with your item bank. Here is an example that can be generalized in many ways. This example quickly shows how to build a form with the following constraints:

  • Build a form with 20 total items
  • Maximize TIF at \(\theta=-2\).
  • Include items from different content strands. This example has four strands and we'll constrain the form to include five items from each strand
  • Constrain the form to include 15 binary items and five polytomous items
  • Constrain the average p-value on the form to be .6. Note, we'll choose 20 items, so the constraint will be the sum of the p-values such that \(p_1 + p_2, \ldots, p_{20} = 12\) given that \(12/20=.6\)
  • Define item enemies so that those enemies do not appear on the same form
  • Create an item group so that at least one intact grouping of items are chosen together

You can replicate this example by downloading the following three files. First you need the item bank file . Second, you will need the constraint definition file and last you will need the initial form file .

For now let's just jump right into things, but see this blog post for descriptions of the ATA module and what these files are.

Step 1: Read In Bank and Base Form Files

We begin by heading over to PACER's Test Construction module and read in the item bank file and the base form file as shown in the screen shot below.

Step 2: Choose Blueprint Dimensions

The next step is to navigate to the add form tab as shown in the screen shot below and choose "Build with test constraints".

Now use the drop down menu "select blueprint dimensions" and choose the following variables as shown here. This is telling PACER which variables hold the metadata associated with each item in the bank we need in order to build the form. If you open the bank file you will see row 1 contains item 1 and it is on strand 1, is binary, has a p-value of .23 and so on for all items.

Step 3: Read in Constraints Definition File

Now we just need to read in the constraint definition file as shown below. These constraints become the definitions passed to the solver used to construct the test form.

Step 4: Build Form, Check It, and Rebuild

Now you simply click on the Build Form and a test form will be created. As soon as you do this the plots will be updated with the new form as you can see in this screen shot.

You can head over to the Form Items tab and see which items were chosen for this new form. These are appearing under Form 3 in this example. Note, we see that items 6, 7, and 8 were chosen together as a group as we hoped. But, how can we check the other test constraints?

If you head over to the "plots tab" again you will see at the bottom there is a frequency report and you can view the number of items chosen by constraint. Here we see 15 binary items were chosen and five polytomous items were chosen exactly as we expected.

Suppose we want to change a constraint. We can head over to the constraint tab and double click the cells in the table as shown below. You can edit the values here and rebuild a test form without having to read in a new spreadsheet.

Some Notes

This quick example shows a rather generalizable way to build test forms with constraints. You can expand this example in many ways to create as many constraints as you wish.
