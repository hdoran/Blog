Classification Accuracy

Classification accuracy helps us understand the degree to which examinees are properly placed into a test category level. If true scores were known, there would be no misclassification at all. Of course, true scores are latent and we only have an estimate of performance referred to as the observed score and that observed score has some amount of measurement error that would lead to some possible misclassifications. So, as the reliability of the measure tends towards 1, we expect fewer misclassifications into groups and as the reliability of the measure tends towards 0, we would expect more misclassification. PACER provides tools for examining the classification accuracy as well as the classification consistency, although this blog post will focus only on classification accuracy based on Rudner's approach . Download this file to your computer and you'll be able to replicate the analyses done in this tutorial.

Read in and Preview Data

The first thing we will do is head over to "Classification Accuracy" in the top navigation bar and read in our data using the browse button to locate the file. The sample data look like the following but there is no requirement at all for data structure, PACER requires only two variables and optionally a third.

The first variable called "score" is the observed score for the examinee and the second variable called "sem" is the standard error of measurement for the individual examinee. These are the only two required variables for this analysis and they do not need to be called "score" or "sem" as we will later see. The third variable called "pl" is the performance level the individual is placed into. We can use this variable in the analysis or PACER can compute the cuts for you. The advantage is that using a precomputed variable will make PACER a bit faster since it has fewer computations to do if you provide this variable--but it's not that big of a deal.

With the data now read into PACER, go to the "options" tab and set up the analysis as follows. Use the drop down menu to choose which variable in your data has the observed score and choose which variable in the data is the standard error of measurement. Now, under "Compute Cuts" use the drop down and choose "FALSE" for now. When you do this, a drop down will appear and you can choose which variable in your data represent the performance level (or group) variable. Last, and very importantly, enter the cut scores used for performance level assignment. If you have \(K\) levels, then you will have \(K-1\) cut scores. Just type in each cut separating them with a comma.

Now, click "Run Classification" and in the main panel you will see the following:

These cards present the entire results in one clean preview and the data can be exported using the buttons over each table. The results are rather comprehensive giving the overall classification accuracy as well as the accuracy at each specific cut as well as the observed counts within each performance level.

Some Notes

If you set "Compute Cuts?" to "TRUE", then you do not need to enter in a performance level variable. PACER will first assign a performance level to each examinee in the file based on the cut scores you enter in the "Cut Scores" box and then it will compute the classification accuracy.
