How To Equate Test Items using PACER

Published January 18, 202210 min read

Test Equating

Test equating is the process of placing scores from different test forms onto a common, comparable scale. This is a common task in psychometric practice and there are many ways to equate test forms. PACER offers six IRT-based approaches including Stocking-Lord, symmetric and non-symmetric approaches, Haebara, symmetric and non-symmetric, Mean/Sigma, and Mean/Mean. The language used in PACER is "bank" items and "operational" or "to link" items. The "bank" items are on the scale we wish to link to. The "operational" or "to link" items need to be linked by estimating a set of linking constants and using those to transform the item parameters onto the bank scale. By way of preference, I do not like the term "linking constant" because the parameters we estimate are random variables subject to sampling variance and they are not constants, although they are often treated as though they are fixed variables.

The items in the linking set can be all binary, all polytomous, or a mixture of the two. All binary items are treated as a special case of the 3-parameter logistic model and the polytomous items can be the generalized partial credit model, or a special case of that model (e.g., Master's Partial Credit) or the graded response model. I will not go into the math in this post, but the Stocking-Lord and Haebara are based on minimization of an objective function where an integral is evaluated using adaptive Gauss-Hermite quadrature. The default is to use 20 quadrature points, however that number can be increased to achieve better approximations as needed.

In this blog post I will show how to use PACER to equate test forms, examine the visual displays associated with equating, and show how to grab the rescaled item parameters. There are many features in the test equating module in PACER and so this blog post will provide sample data and just get you up and running with test equating. Other future posts will show some of the other features.

Preparing the Data

In order to run the test equating, we need two sets of item parameters. Data file 1--the bank parameters can be downloaded here and the to link parameters can be downloaded here. The data in each file must look like this sample:

Here we see the variable names ITEM_ID, SCOREPOINTS, and then P0, P1, P2, .... For binary items (SCOREPOINTS =1), the columns mean P0 = a-parameter, P1 = b-parameter, and P2 = c-parameter. For polytomous items, the columns have a slightly different meaning. When SCOREPOINTS > 1, then P0=a-parameter, however, the columns P1 through P3 are the step values indicating where the curves of the GPCM intersect. By the way, there can be more columns for step values. Just label them P4, P5, etc.

Some IRT software programs represent the GPCM as \((\theta - (\delta_j - b_{j,k}))\) where \(\delta_j\) is a location parameter and \(b_{j,k}\) is a category and \(\sum_{k=1}b_{j,k}=0\). In PACER, simply transform this to be \((\theta - \lambda_{jk})\) where \(\lambda_{jk} = (\delta_j - b_{j,k})\).

Running the Analysis in PACER

Now with our data formatted, we can easily run the analysis using PACER. First, click the "Test Linking" tab in the top navigation bar and you'll end up at this screen where you simply read in the data as you would any file using browse to find the file. The bank file has the parameters on the scale we want and the toLink parameters will be linked to the bank scale.

You know your data are properly read in when you see "Upload Complete". Now, all you need to do to run the analysis is move over to the "Options" tab and use the drop down menu to choose your method and choose a few options.

Now choose a few optional settings, like the number of quadrature points. As mentioned above, PACER uses adaptive Gauss-Hermite and the default is 20 quadrature points. You can increase this and test out more points and see if the estimates change.

Now all you need to do is click "Run Equating Modules" and PACER will begin estimation. A progress bar will appear in the lower right corner of your screen and when the analysis is complete you will see the following summary printed to screen.

The summary text provides the estimates--referred to as A: Slope and B: Intercept and some diagnostics. An important diagnostic is the \(D^2\) statistic, a measure of distance between the ICC curves. The diagnostic prints the five items with the largest \(D^2\).

Now we can move over the the "Visual Displays" tab and examine the ICCs and the TCCs. First, note that the top plot is the two ICCs for the selected item. Use the drop down menu to choose a different item. The bottom plot shows two TCCs. Well, there are two, but they are almost right on top of each other in this example, so it seems like there might be only one.

Now if you head over to the "Item Parameters" tab can see the data table of the to link items rescaled using the estimates obtained from the equating process. This is a searchable and filterable table and you can use the buttons on top to export or print the table as you wish. The column D2 is the \(D^2\) for each item and the column SD2 is the standardized \(D^2\).

For the most part, that's all there is in order to equate items. But, there are some additional features to document. First, suppose you want to rerun the equating with some items removed from the set. Simply click in the "Select Items to Drop" box and choose some items. When you're done, simply click "Run Equating Modules" again and the analysis will be rerun and will now exclude those items from the analysis.

I'll describe one option for now and leave the other options for another blog post. If you choose the Stocking-Lord or Haebara approaches, you will see an option to choose the population parameters. When a symmetric approach, you will see inputs for the bank population and another for the OP (or to be linked) population. The idea of the symmetric approach is we can define the two different populations from which the examinees are drawn. If a non-symmetric approach is selected, you would only see bank population.

Experiment with the drop down menus and choose different methods, try dropping items, and try different numbers for the quadrature points. For the most part, the experience here should be intuitive if you're familiar with test equating. Future blog posts will explain the other options and how they can be used.
