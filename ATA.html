Automated Test Assembly

Automated test assembly (ATA) is a powerful technique for building test forms. It replaces the manual process of humans choosing test items to meet specific criteria and allows an algorithm to choose items instead. This blog post shows how to quickly get up and running with the test construction features in PACER including ATA. It's easiest to download the sample files and replicate the tutorial as shown below for the best experience and to see how all files are structured.

In the terminology of ATA the term "constraints" are features of the items or test we want the form to adhere to. PACER provides a platform to use linear programming to build a complex test form with many test constraints via the lpSolver API. However, PACER goes beyond simple test construction and puts all the tools a test developer and psychometrician needs in one place.

Specifically, the test construction module in PACER allows you to quickly view TCC, TIF, and SEM plots, view a score conversion table, estimate marginal reliability of the form, and build test forms all in the browser. This is extremely convenient as many things are available in a simple point and click environment all found in one location.

This blog post will show how to use the test construction module including the ATA engine. It is intended to be a quick tutorial to get you up and running with the ATA options in PACER. The example is simple to show how to use the module, but the number of test constraints can be much larger than those demonstrated here. To follow along with the example, download the sample files including the item bank file . Then, you will need a second file that lists the items you want to incude on a form here and last to use ATA you need a file that defines your test constraints like this one here .

Data Structure Explanation

We need three specific data files to get up an running with the ATA options in PACER. The first file is the parameters file and is structured as follows below. This file holds the item parameters and any features associated with those items we might later use as test constraints. Download the bank_poly.csv file in the link above to see this complete file.

The required variables are ItemID, ScorePoints, Model, and the P0-P10 columns, which I will explain. First, this file must hold all of the item parameters that will be used in the test construction process. The scorepoints variable tells PACER how many scores are expected for that item (e.g., multiple choice is 1). The model variable for binary items is always IRT3pl. All binary items are treated as a special case of the 3PL.

For example, the Rasch model has an a-parameter set to 1 (P0), a location parameter (P1), and then a c-parameter set to 0 (P2). In PACER, column P0 holds the a-parameters for an item. Column P1 holds the b-parameter, and column P2 holds the c-parameter. So, this also shows PACER can score a mixture of item types as well. Some of the item might be 2pl, some might be 1PL, some might be 3PL. The scoring system would score accordingly just by passing it the types of item parameters item needs.

You can have a mixture of polytomous and binary items too. This file shows how that might be structured. Notice, the first item is setup as a 3PL item and the fourth item is setup as a generalized partial credit item. It has 2 scores points. For this GPC item, P0 is its a-parameter, P1 is step value 1 and P2 is step value 2. Note, if you want the GPC to actually be Master's partial credit, then P0 would just be 1! So, PACER views items as special cases of the more general case making it very easy.

The second file you need is the item list file and is structured as shown in the screen shot below. You can also download this file as the lpitems.csv file in the link above. This file has two columns meaning we want to build two test forms with the items in each column. This file must have at least one column and at least one item but can have as many columns as you wish after that. PACER will build a test form for each column and the values in the column are the item IDs. That is, column 1 holds the item IDs you wish to include in form 1, column 2 has the items in form 2 and so on. Note, these item IDs must be in the parameter file.

The third and final file for the ATA option defines the constraints we wish to use when building our new form. The first constraint is required and tells PACER the total number of items to include on the test form. The additional constraints are all optional and are based on the features available for the items in the parameters file. Note above in the parameters file we have two columns called "strands1" and "strands2". They can be called something else, but these are the features associated with the items we wish to use as test constraints.

These constraints provide the instructions on how to build the form. Here we want 20 total items and we want 5 items from strand 1, 5 from strand 2 and so on. In addition, we want the form to consist of 15 binary items and 5 polytomous items.

Let's Build Some Forms

The first step is to head over to PACER under the Test Construction tab and read in those two files as shown in the screen shot below. PACER will automatically build two forms using the items listed in the items file and will immediately print two TCCs to the screen for each of those forms.

Notice, you can simply move over to the Options tab and click on TIF and new test information plots will immediately be printed to the screen for each form.

So far PACER has only built forms using the items you read in. We can now head over to the "Add Form" tab and build some new forms using one of three options. The first option is to manually choose items using the drop down menu as shown in the screen shot below. As soon as you begin choosing items and plot will be added to the screen representing that new form and updated in real time.

A second option is to use the quick build feature as shown below. Here we simply build a test form that maximizes the TIF at theta = -1 and we tell PACER to choose 20 test items. Immediately the items will be chosen and a plot will be printed to screen immediately. It's important to point out that you can also visit the other tabs as soon as you have a form to view the score conversion table or the reliability of the form as well.

A third option is to use the build with test constraints feature as shown below. Here we need to set up a few more things in order to properly use ATA. First, use the drop down menu called "Select blueprint dimensions" and select the columns in the parameters file that hold the features associated with the items you wish to use for the ATA process. In this sample, we choose strand1 which has items in one of four strands and a second column called strand2 which denotes if an item is binary or polytomous. Next let's read in the file called constraints_poly.csv. This has the instructions that are passed to PACER describing specifically how to build the form. Once you read those in click Build Form and a test form meeting those constraints (if possible) will be built and a plaot will be printed to screen. You can choose new values of theta in the browser to meet a form with the same constraints but at a different value for theta.

Of course, we want to know if the ATA worked to produce a test form that met those constraints. Notice at the bottom of the panel is a card printing the frequency report. If we use the drop down and choose the variable called strand2 for example we see the form consists of 15 binary items and 5 polytomous items exactly as we expected. You can use the drop down to explore the other features and constraints you might wish to view and the table will be immediately updated.

Other Available Features

Notice the tabs at the top offer many other features you can explore. For instance, if you head over to the Form Items tab you will see a table showing which items belong to each form. This can be easily exported using the buttons above the table.

Or you can head over to the score conversion tab and immediately get a score conversion table for the form you built. Notice by simply clicking on MAP or EAP for example the score conversion table will update using that method for scoring.

Some Notes

This is intended as a quick guide to get you up and running with test form construction methods in PACER. Many other features are available and we'll explore those in future blog posts.
