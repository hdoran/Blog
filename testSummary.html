How To Create Test Summary Plots and Score Conversion Tables Using PACER

Published January 22, 20225 min read

Test Summary Plots and Score Conversion Tables

Visual displays of test forms and test score conversion tables are two very standard needs for psychometricians. PACER is a complete IRT-based test environment and its capacity for test scoring will be fully described in another blog post. For now we explore the overall summaries of test forms through visual displays and with general score conversion tables often used for test scoring or reviewing the test before launching into a full test scoring routine.

In this blog post I will walk through a simple and effective way to get test information functions, standard error plots, test characteristics curves, score conversion tables and the marginal reliability of test forms. This requires only two data files and samples can be downloaded for practice and replicating this tutorial. See links below in next section.

Preparing the Data

In order to generate the visual displays we require two data files. The first file holds the item parameters and a sample to run this tutorial can be donwloaded here. The second file is a list of the item identifiers you wish to include in these summaries a sample of which can be downloaded here. Let's examine these two files so their structure is understood.

First, a snapshot of the parameters file is below. For this example, it's a simple structure Here we see the variable names ITEM_ID, SCOREPOINTS, and then P0, P1, P2, .... For binary items (SCOREPOINTS =1), the columns mean P0 = a-parameter, P1 = b-parameter, and P2 = c-parameter. For polytomous items (although there are none in this example), the columns have a slightly different meaning. When SCOREPOINTS > 1, then P0=a-parameter, however, the columns P1 through P3 are the step values indicating where the curves of the GPCM intersect. By the way, there can be more columns for step values. Just label them P4, P5, etc.

If you have polytomous items and want to learn more about how to include them in a parameters file see this blog post Test Equating for some additional details. The second file is nothing more than a list of the items you want to include and looks like this:

First notice we have three columns which will tell PACER we will plot three test forms. There can be one column, two columns, or any number. The rows are simply the item ID of the items you want to include. So, column 1 for example begins with item 106294, then 5199 and so on. The item IDs must be item IDs that exist in the parameters file and they can have any form (numeric, character, etc). The second column has different items and also has fewer items that column 1. So, we can include any number of items in these columns. This might be useful to create a plot or score conversion table of parallel forms or of one form with strands within the overall form containing a subset of the items.

Running the Analysis in PACER

Now with our data formatted, we can easily run the analysis using PACER. First, click the "Test Summary" tab in the top navigation bar and you'll end up at this screen where you simply read in the data as you would any file using browse to find the file.

Now in this case as soon as you load both data files a test information plot (TIF) is immediately drawn on the screen for all three test forms we are including. We can now head over to the "Options" tab where we can simply use the drop down menu and select a different type of visual display for our summary. These will be immediately drawn when you choose an option.

Now head over to the "Score Options" and the "Score Conversion" tabs and you will immediately see a score conversion table printed to the screen. Notice at the bottom of the side panel you will see "Choose a column for the score conversion table" and in this example it is a 1. That means it generated a score conversion table for the items in the first column of your items file. If you simply change this to a 2, it will create a score conversion table using only the items included in column two of your items file.

Also notice we can choose different scoring options. Experiment with the options and watch as a new score conversion table is immediately created. If you choose one of the two Bayesian options (MAP or EAP), you will see an option to choose the population mean and variance. The two frequentist methods (MLE and TCC) are pretty commonly used.

Some Notes

You will also see there is a tab for the "marginal reliability". Head over to that tab and you will see the marginal reliability estimated for the test forms you select. This is estimated using only the item parameters and it integrates out the nuisance (theta) parameter. Also, things are designed to be simple and intuitive in this environment. Note that you can export the score conversion tables, export the plots, change options for various things, all in a simple way. The best thing is to mess around with the different options and watch what happens to the plots or the data tables as you do so.
